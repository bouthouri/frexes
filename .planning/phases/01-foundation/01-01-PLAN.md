<task type="auto">
  <name>Task 1: Scaffold Astro project with React integration</name>
  <files>package.json, bun.lock, astro.config.mjs, tsconfig.json, src/pages/index.astro</files>
  <action>
    1. From the project root (`/Users/bouthouri/work/frexes`), create the Astro project:
       ```bash
       bun create astro@latest . --template basics --install --git --no
       ```
       (Use `--no` to skip prompts if supported, otherwise accept defaults. The project root already has .git so use existing.)

    2. Add React integration:
       ```bash
       bunx astro add react --yes
       ```

    3. Install animation libraries:
       ```bash
       bun add motion gsap
       ```

    4. Install Tailwind v4 via vite plugin (NOT @astrojs/tailwind):
       ```bash
       bun add tailwindcss @tailwindcss/vite
       ```

    5. Update `astro.config.mjs` to include Tailwind v4 as a Vite plugin:
       ```js
       import { defineConfig } from "astro/config";
       import react from "@astrojs/react";
       import tailwindcss from "@tailwindcss/vite";

       export default defineConfig({
         integrations: [react()],
         vite: {
           plugins: [tailwindcss()],
         },
       });
       ```

    6. If `tailwind.config.*` was auto-generated, DELETE it — Tailwind v4 uses CSS-first config only.

    7. Verify `src/pages/index.astro` exists (created by template). Replace its content with a minimal page that imports global CSS:
       ```astro
       ---
       ---
       <html lang="en">
         <head>
           <meta charset="utf-8" />
           <meta name="viewport" content="width=device-width, initial-scale=1" />
           <title>Frexes</title>
         </head>
         <body class="bg-white text-foreground">
           <main class="flex min-h-screen items-center justify-center">
             <h1 class="text-4xl font-bold">Frexes</h1>
           </main>
         </body>
       </html>
       ```

    IMPORTANT: Do NOT install @astrojs/tailwind — that's the deprecated v3 integration.
    IMPORTANT: Do NOT create tailwind.config.js/ts — Tailwind v4 uses CSS @theme directives.
  </action>
  <verify>
    <automated>cd /Users/bouthouri/work/frexes && bun build 2>&1 | tail -5</automated>
  </verify>
  <done>Astro builds without errors. package.json contains astro, @astrojs/react, tailwindcss, @tailwindcss/vite, motion, gsap.</done>

<task type="auto">
  <name>Task 2: Initialize shadcn/ui and configure Tailwind v4 theme tokens</name>
  <files>components.json, src/lib/utils.ts, src/styles/global.css</files>
  <action>
    1. Initialize shadcn/ui for Astro:
       ```bash
       bunx shadcn@latest init -t astro
       ```
       Follow prompts: TypeScript, default style, neutral base color. This creates `components.json`, `src/lib/utils.ts` (with `cn()` utility), and `src/components/ui/` directory.

    2. If shadcn init created a `tailwind.config.*` file, DELETE it — not compatible with Tailwind v4.

    3. Create or update `src/styles/global.css` with Tailwind v4 CSS-first config:
       ```css
       @import "tailwindcss";

       @theme {
         --color-background: #ffffff;
         --color-foreground: #0a0a0a;
         --color-muted: #f5f5f5;
         --color-muted-foreground: #737373;
         --color-border: #e5e5e5;
         --color-accent: #000000;
         --font-sans: "Inter", ui-sans-serif, system-ui, sans-serif;
         --font-heading: "Inter", ui-sans-serif, system-ui, sans-serif;
       }
       ```

       Using Inter for both body and headings — geometric, clean, matches the bold favicon aesthetic and Vercel-inspired design direction. (Per Claude's discretion on typography.)

    4. Ensure `src/pages/index.astro` imports the global CSS. Update the `<head>` to include:
       ```astro
       ---
       import "../styles/global.css";
       ---
       ```

    5. Verify `cn()` utility works by checking `src/lib/utils.ts` exists and exports `cn`.

    NOTE: shadcn/ui init for Astro may have specific quirks with Tailwind v4. If init fails or generates v3-style config, manually create `components.json` and `src/lib/utils.ts` with the cn() utility:
    ```ts
    import { clsx, type ClassValue } from "clsx";
    import { twMerge } from "tailwind-merge";

    export function cn(...inputs: ClassValue[]) {
      return twMerge(clsx(inputs));
    }
    ```
    And install: `bun add clsx tailwind-merge`
  </action>
